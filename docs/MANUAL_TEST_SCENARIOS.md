# 手動テストシナリオ（ローカル画面操作）

ローカルの Firebase Emulator + シードデータで、主要画面を徹底的に操作確認するための手順書。

## 0. 事前準備

```bash
# 1) エミュレーター起動（ターミナル1・起動しっぱなし）
firebase emulators:start --only auth,firestore
#   データを残したい場合:
#   firebase emulators:start --only auth,firestore \
#     --import=./emulator-data --export-on-exit=./emulator-data

# 2) シード投入（ターミナル2・初回は scripts で npm install firebase-admin）
cd scripts && npm install firebase-admin && cd ..
node scripts/seed_test_data.js          # ユーザー+車両3台+整備記録を投入
#   作り直したいとき: node scripts/seed_test_data.js --reset

# 3) アプリ起動（ターミナル3）
flutter run -d chrome    # または -d macos（127.0.0.1 接続に修正済み）
```

ログイン: `test@example.com` / `test1234`

> 接続先は `lib/main.dart` で `127.0.0.1` に修正済み（macOSのIPv6問題回避）。
> エミュレーターを再起動するとデータは消えるため、その都度シードを再投入する（または `--import/--export-on-exit` を使う）。

投入される車両（手動確認の起点）:
| 車両 | 特徴 | 確認ポイント |
|------|------|------------|
| トヨタ カローラ（ハイブリッド） | 車検まで約120日 | 標準ケース |
| ホンダ N-BOX（軽） | **車検まで約25日** | 車検リマインダー/警告表示 |
| トヨタ ハイエースバン（貨物） | 用途=貨物（毎年車検）、走行98,000km | 用途区分による車検サイクル差 |

## フロー1: 認証まわり

- [ ] ログイン画面が表示される（TrustCarロゴ / メール / パスワード / ログイン / 新規登録）
- [ ] 不正なメール形式でバリデーションエラーが出る
- [ ] 空のパスワードでバリデーションエラーが出る
- [ ] 正しい資格情報（test@example.com / test1234）でログイン → ホームへ遷移
- [ ] 誤ったパスワードでエラースナックバーが出る
- [ ] 「パスワードを忘れた場合」→ メール未入力時に警告
- [ ] ログアウト → ログイン画面に戻る

## フロー2: 車両一覧 → 詳細

- [ ] ホーム/車両一覧にシードの3台が表示される
- [ ] 各車両カードにメーカー・車種・ナンバー・走行距離が表示される
- [ ] N-BOX に車検期限間近の警告/バッジが出る（残り約25日）
- [ ] 車両をタップ → 詳細画面に遷移
- [ ] 詳細に型式・色・排気量・燃料・車検満了日などが正しく表示される
- [ ] ハイエース（貨物）の車検サイクルが乗用車と異なって扱われている

## フロー3: 車両登録（新規追加）

- [ ] 「車両追加」から登録画面を開く
- [ ] 必須項目（メーカー・車種・年式・グレード・走行距離）未入力でバリデーション
- [ ] 全項目入力して保存 → 一覧に4台目が増える
- [ ] 追加直後にエミュレーターUI(localhost:4000)の `vehicles` に1件増えていることを確認

## フロー4: 整備記録

- [ ] 車両詳細から整備履歴一覧を開く → シードの整備記録が日付降順で並ぶ
- [ ] 整備記録の詳細（種別・費用・店名・走行距離）が表示される
- [ ] 新規整備記録を追加（種別=オイル交換、費用、日付、走行距離）→ 一覧に反映
- [ ] 車検記録（type=carInspection）で車検結果や費用内訳（部品/工賃/諸費用）が表示される
- [ ] 費用の合計表示・期間フィルタ（あれば）の動作確認

## フロー5: 通知 / リマインダー

- [ ] 車検リマインダーに N-BOX（期限間近）が出る
- [ ] 通知一覧の表示・既読操作
- [ ] 走行距離ベースの交換推奨（nextReplacementMileage）があれば表示確認

## フロー6: 検索・その他画面

- [ ] 整備記録/車両の検索・絞り込み
- [ ] 店舗一覧（`scripts/seed_shops.js` を投入していれば）と近い順ソート
- [ ] プロフィール画面の表示・編集
- [ ] オフライン/再接続時の挙動（任意）

---

## 不具合を見つけたら

1. `flutter run` コンソールのエラーログを控える（特に `[cloud_firestore]` / `[firebase_auth]`）
2. エミュレーターUI `http://localhost:4000` でデータの実体を確認
3. 再現手順とともに Issue 化（ラベル: `bug,claude-task`）

## トラブルシュート早見

| 症状 | 原因 | 対処 |
|------|------|------|
| ログインボタンが無反応（固まる） | エミュレーター未起動 or 接続先不一致 | 9099/8080 の起動確認、`127.0.0.1` 接続を確認 |
| ログイン後に一覧が空 | シード未投入 or 別UID | `node scripts/seed_test_data.js` を実行 |
| `EMAIL_EXISTS` | ユーザー作成済み | そのままログインでOK |
| 起動毎にデータが消える | エミュレーターは揮発 | `--import/--export-on-exit` を使う |
