## エグゼクティブサマリー

Trust Car Platformは「クルマにまつわるすべての不安を、一つのアプリで解消する」サービスです。  
本書では、実際のユーザー像（ペルソナ）9種類を定義し、それぞれの**課題・行動パターン・利用シナリオ・成功指標**を体系的に記述します。

## Persona A：個人オーナー（4台混在ガレージ）

### プロフィール

| 項目 | 内容 |
|------|------|
| 年齢・属性 | 40代男性・自営業 |
| 居住地 | 東京都品川区 |
| 車両構成 | アルファード（乗用）・ハイエース（貨物）・ノート（日産リース）・ロードスター（スポーツ） |
| テクリテラシー | 普通（スマホは使いこなしている） |
| 年間整備費 | 推定40〜60万円 |

### ペインポイント

1. **ハイエース（貨物車）は毎年車検**。乗用車と同じ感覚でいると気づかずに切れる
2. **リース車のノートには契約満了日**がある。いつ返すべきか分からなくなる
3. **ロードスターは長期保管が多く**、車検日を登録していない
4. 4台分の車検日を手帳で管理していて煩雑。一つのアプリで見たい

### 利用シナリオ

```
[毎月1日] アプリを開いてホーム画面の「車検・点検アラート」を確認
         ↓
[ハイエースが30日以内に車検期限] → クリックで整備工場に一括問い合わせ
         ↓
[問い合わせ文面を自動生成] → 「品川 400 か 22-22（貨物車/毎年車検）
         が○月○日に車検期限を迎えます。ご連絡お願いします」
         ↓
[工場から返信] → アプリ内で日程調整
```

### 利用機能マップ

| 機能 | 利用頻度 | 重要度 |
|------|---------|--------|
| 車両一覧・ステータス表示 | 毎週 | ★★★ |
| 車検残日数カウントダウン | 毎月 | ★★★ |
| 車両用途区分（貨物/乗用） | 登録時のみ | ★★★ |
| リース満了日管理 | 毎月 | ★★ |
| 一括問い合わせ文面生成 | 年2〜4回 | ★★★ |

### 成功指標

- 貨物車の車検切れ件数: ゼロ
- 問い合わせ送信時間: 3分以内
- リース満了の見落とし: ゼロ

## Persona B：中小企業フリート管理者（20台一括管理）

### プロフィール

| 項目 | 内容 |
|------|------|
| 年齢・属性 | 30代女性・総務担当（田中 花子） |
| 会社規模 | 従業員50名・運送業 |
| 車両構成 | 貨物車10台・乗用車10台（全20台） |
| 担当者アサイン済み | 10台（各ドライバーに1台） |
| 年間整備費 | 推定200〜300万円 |

### ペインポイント

1. **20台のうち車検が危ない車両をすぐに把握したい**（Excelで管理中・更新ミスが多い）
2. **整備工場への問い合わせが台数分バラバラ**になっていて非効率
3. **CSVエクスポートで経営会議に報告**したいが、手動集計に1時間かかる
4. **担当ドライバーが勝手に工場を決めてしまう**。管理者が関与できない

### 利用シナリオ

```
[月初] ダッシュボードを開く → フリート統計を確認
       → critical（車検切れ・7日以内）: 3台
       → warning（8〜30日以内）: 3台
       → normal: 14台
         ↓
[critical 3台を選択] → 一括問い合わせ文面を生成
「【緊急】車検期限が近い車両が6台あります。
  ・品川 400 え 01-01（貨物・期限切れ5日前）
  ・品川 400 え 02-02（貨物・期限切れ5日前）...」
         ↓
[CSVエクスポート] → ナンバー・担当者・車検日・緊急度を含む全20行を出力
         ↓
[月次報告] Excelに貼り付けて経営会議へ提出
```

### 利用機能マップ

| 機能 | 利用頻度 | 重要度 |
|------|---------|--------|
| フリート統計ダッシュボード | 毎日 | ★★★ |
| 緊急度別分類（critical/warning/normal） | 毎日 | ★★★ |
| 一括問い合わせ文面生成 | 月2〜4回 | ★★★ |
| CSVエクスポート | 月1回 | ★★ |
| 担当者アサイン・権限管理 | 随時 | ★★ |
| フリートコード参加 | 入社時 | ★ |

### 成功指標

- 車検切れゼロ（critical=0 の状態を維持）
- 報告書作成時間: 1時間 → 5分
- 整備工場への問い合わせ漏れ: ゼロ

## Persona C：整備工場比較ユーザー

### プロフィール

| 項目 | 内容 |
|------|------|
| 年齢・属性 | 35歳女性・会社員 |
| 居住地 | 世田谷区 |
| 車両 | Honda フィット（5年目） |
| テクリテラシー | 高め（比較サイト利用経験あり） |
| 決め手 | 口コミ評価・得意サービス・自宅からの距離 |

### ペインポイント

1. **近所の整備工場が3つあるが、どこが何得意か分からない**
2. **評価が高い工場が近いとは限らない**
3. **車検はA工場・カスタムはB工場**と使い分けたいが、情報が散らばっている
4. 新規工場は口コミが少なくて判断しにくい

### テスト用工場データ（Persona Cのシナリオ）

| 工場名 | 得意 | 評価 | レビュー数 | 距離 |
|--------|------|------|---------|------|
| 車検のスピード太郎 世田谷店 | 車検・整備 | 4.8 | 120件 | 約1km |
| ガレージ ワークス | カスタム・板金 | 4.2 | 45件 | 約3km |
| トヨタモビリティ サービス | 車検・タイヤ | 4.0 | 8件 | 約5km |

### 利用シナリオ

```
[車検前3ヶ月] 工場比較画面を開く
       → 「車検」フィルタで2工場が候補に絞られる
       → 評価: スピード太郎(4.8) > モビリティ(4.0)
       → 距離: スピード太郎(1km) < モビリティ(5km)
       → AIスコア = 評価×ln(レビュー数) - 距離ペナルティ
             スピード太郎: 4.8×ln(121) - 0.05 = 4.8×4.79 - 0.05 = 22.95
             モビリティ: 4.0×ln(9) - 0.25 = 4.0×2.20 - 0.25 = 8.55
         ↓
[スピード太郎を選択] → 問い合わせ送信
```

### 利用機能マップ

| 機能 | 利用頻度 | 重要度 |
|------|---------|--------|
| 工場サービス種別フィルタ | 比較時 | ★★★ |
| 評価・レビュー数表示 | 比較時 | ★★★ |
| 距離ソート（Haversine） | 比較時 | ★★ |
| AIスコアランキング | 比較時 | ★★★ |
| 問い合わせ送信 | 工場選定後 | ★★★ |

### 成功指標

- 工場選定時間: 30分 → 5分
- 「近所の工場なのにはずれだった」のゼロ化

### ペインポイント

2. **ナンバーや個人情報が記録に残ることへの不安**

### 利用シナリオ

```

       → 理由: 「売却（sold）」
       → データ保持: 「はい（isDataRetained=true）」
       → メモ: 「ガリバー買取 35万円」
         ↓
[退役後] 車両が「過去の車両」タブに移動
       → 状態バッジ: 「売却済み」（黄色）
       → 整備記録はそのまま閲覧可能
       → ナンバー・担当者情報は任意削除可
         ↓
[もし誤操作なら] 「復元する」ボタン → 確認ダイアログ → 使用中に戻る
         ↓
[次の車購入時] 「過去の車両」タブで10年分の整備記録を参照
               → 「前の車はタイヤ交換を2年毎にしていた」を参考に新車の整備計画を立てる
```

## Persona G：EVオーナー（日産リーフ）

### プロフィール

| 項目 | 内容 |
|------|------|
| 年齢・属性 | 38歳男性・エンジニア |
| 車両 | 日産リーフ（2022年〜・25,000km） |
| EV固有の特徴 | オイル交換不要。タイヤ・ブレーキフルード・ワイパーが主な整備項目 |
| 関心 | EV特有の整備パターンを同じEVオーナーと共有したい |

### ペインポイント

1. **「EV向け整備アプリがない」**。ガソリン車向けの案内がノイズになる
2. **EVのタイヤはローテーションが早い**（回生制動で前輪摩耗）。同様の悩みを持つ仲間を探したい
3. **ブレーキフルードの交換頻度が分からない**（EV特有のブレーキ特性）
4. コミュニティに「オイル交換の質問」が多くて、EV関連の情報が埋もれる

### 利用シナリオ

```
[EVオーナー登録] 燃料種別: EV / 車種: 日産リーフ
         ↓
[整備記録入力] タイヤ交換・ブレーキフルード・ワイパーのみ入力
               ← オイル交換の入力欄はあるが「0件」でも正常動作
         ↓
```
