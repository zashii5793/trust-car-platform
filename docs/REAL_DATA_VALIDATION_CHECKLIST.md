## 参考情報

- 車検証電子化: 2023年1月〜（普通車）、2024年1月〜（軽自動車）  
  出典: [国土交通省北信越運輸局](https://wwwtb.mlit.go.jp/hokushin/00001_00829.html)
- 新様式: A6サイズICカード + 「自動車検査証記録事項」A4別紙の2点セット  
  出典: [電子車検証特設サイト](https://www.denshishakensho-portal.mlit.go.jp/)
- Google MLKit日本語対応: `TextRecognitionScript.japanese` で漢字・ひらがな・カタカナ認識  
  出典: [pub.dev google_mlkit_text_recognition](https://pub.dev/packages/google_mlkit_text_recognition)
- 整備請求書標準項目: 品目・数量・単価・部品代・工賃・消費税10%・合計  
  出典: [マネーフォワード クラウド請求書](https://biz.moneyforward.com/invoice/basic/55085/)

## Phase 1: 車検証OCR

### 準備物
- 旧様式（〜2022年）A4車検証 × 実物または鮮明な写真
- 新様式（2023年〜）ICカード（A6）＋「自動車検査証記録事項」（A4別紙）× 実物
- スマートフォン（iOS / Android 各1台）
- 明るい屋内照明（直射日光を避ける）

### シナリオ一覧

| # | シナリオ | 様式 | 期待精度 | 確認項目 |
|---|--------|------|--------|---------|
| V-01 | トヨタ系（DBA-/ZN6- 型式）旧様式 | 旧A4 | 90%+ | 登録番号・車台番号・型式・車名・有効期間 |
| V-02 | ホンダ系（5AA- 型式）旧様式 | 旧A4 | 85%+ | 全15フィールド |
| V-03 | 日産系（BNR35- 型式）旧様式 | 旧A4 | 75%+ | 車台番号の5文字プレフィックス認識 ※制限あり |
| V-04 | 軽自動車（660cc）旧様式 | 旧A4 | 85%+ | 排気量660cc・乗車定員4人 |
| V-05 | ハイブリッド車（ハイブリッド燃料） | 旧A4 | 85%+ | 燃料の種類=ハイブリッド |
| V-06 | 電気自動車（EV, ZE1- 型式） | 旧A4 | 85%+ | 燃料の種類=電気・排気量なし |
| V-07 | 新様式 ICカード（A6）OCR | 新A6 | 50%+ | ※ICタグ情報はOCR不可・登録番号のみ |
| V-08 | 新様式「記録事項」（A4別紙）OCR | 新A4別紙 | 85%+ | 旧様式と同等情報が記載 |
| V-09 | 斜め撮影（約15度傾き） | 旧A4 | 60%+ | 傾き補正なし・精度劣化を確認 |
| V-10 | 低照度環境（薄暗い屋内） | 旧A4 | 40%+ | 照度不足時の精度劣化・エラーメッセージ確認 |

### 確認手順（V-01〜V-08 共通）

1. アプリ起動 → 「車両を追加」→「車検証をスキャン」
2. カメラが起動することを確認
3. 車検証全体がファインダーに収まるよう撮影
4. OCR処理中スピナーが表示されることを確認（2〜5秒）
5. 「車検証読み取り結果」画面に遷移することを確認
6. 各フィールドに値が入力されているか確認（以下チェック）

### フィールド別確認リスト（V-01〜V-08）

```
[ ] 登録番号（ナンバープレート）: 例「品川 300 あ 1234」
[ ] 車台番号: 例「ZN6-0123456」
[ ] 型式: 例「DBA-ZN6」
[ ] 車名（メーカー）: 例「トヨタ」
[ ] 車名（モデル）: 例「86」
[ ] 初度登録年月（西暦変換）: 例「2013」
[ ] 有効期間の満了する日: 例「2025-05-20」
[ ] 総排気量（cc）: 例「1998」
[ ] 燃料の種類: 例「ガソリン」
[ ] 色: 例「赤」
[ ] 乗車定員: 例「4人」
[ ] 車両重量（kg）: 例「1213」
[ ] 車両総重量（kg）: 例「1323」
```

### 既知制限事項

| 制限 | 詳細 | 回避策 |
|------|------|--------|
| 5文字以上プレフィックスの車台番号 | BNR35-, ZC53S-, ZVW50- 等は末尾4文字のみ取得される | 手動修正 |
| 新様式ICカード（A6） | 所有者情報・有効期間はICタグにあり、OCRでは取得不可 | 「記録事項」別紙をスキャン |
| 和暦変換（昭和） | 昭和40年代等の古い登録年はサポート対象だが精度低め | 手動入力 |
| 斜め・反射 | 精度が顕著に低下 | 垂直・間接光で再撮影 |

### 確認手順（I-01〜I-05 共通）

1. 整備記録追加 → 「請求書をスキャン」
2. 請求書全体を撮影
3. 「整備内容確認」画面で以下を確認

### フィールド別確認リスト

```
[ ] 整備日（日付）: 例「2024-03-15」
[ ] 合計金額: 例「¥7,700」
[ ] 消費税額: 例「¥700」
[ ] 小計（税抜）: 例「¥7,000」
[ ] 店舗名: 例「株式会社カーサービス山田」
[ ] 電話番号: 例「03-1234-5678」
[ ] 整備種別推定: 例「エンジンオイル交換」
[ ] 走行距離（記載あり場合）: 例「45,231km」
[ ] 明細項目（1件以上）
[ ] 車両番号（記載あり場合）: 例「品川 300 あ 1234」
```

### 請求書フォーマット別対応状況

| フォーマット | 対応 | 備考 |
|------------|------|------|
| 令和元年/西暦混在 | ✅ | 両方のパターンを認識 |
| ¥記号付き金額 | ✅ | ¥7,700形式 |
| 「円」表記 | ✅ | 7700円形式 |
| カンマ区切り | ✅ | 7,700形式 |
| 税込/税抜表示 | ✅ | 小計・税・合計の分離 |
| QRコード付き | ⚠️ | QR部分が誤認識される場合あり |

## OCR精度改善ヒント

実データテスト後に精度が低い場合は以下を確認：

1. **撮影品質**: 300dpi相当・影なし・水平撮影
2. **VIN番号5文字プレフィックス**: `vehicle_certificate_ocr_service.dart` の `_extractVinNumber` 正規表現を `[A-Z0-9]{2,6}` に拡張
3. **新様式対応**: 「記録事項」（A4別紙）をスキャン対象とするUI案内を追加
4. **テキスト認識率**: `TextRecognitionScript.japanese` が最適。英語専用スクリプトに変更しない
5. **MLKit自動精度レポート**: `flutter test test/ocr/ --reporter expanded` で実行可能
