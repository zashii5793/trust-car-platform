# Business Plan — Trust Car Platform

> 最終更新: 2026-04-14

---

## プロダクト概要

「クルマをもっと安全に、楽しく、個性的に。」

車両管理・SNSコミュニティ・マーケットプレイスを一体化した
カーライフ・コンシェルジュアプリ。日本の自動車保有台数約8,000万台を市場とする。

---

## ターゲットユーザー（ペルソナ）

| セグメント | 属性 | 悩み・ニーズ | 主な利用機能 |
|-----------|------|------------|------------|
| 一般カーオーナー（メイン） | 20-40代、普通免許保有、車に詳しくない | 車検期限を忘れる、整備記録の管理が面倒 | アラート、整備記録、OCR自動入力 |
| クルマ好き（カーマニア） | 20-40代男性、改造・カスタム趣味 | 愛車を自慢したい、パーツ情報が欲しい | SNS、パーツ提案、ドライブログ |
| DIYメンテ層 | 30-50代、自分で整備を行う | 消耗品交換タイミングの把握、費用記録 | 整備記録、AIパーツ提案、整備統計 |
| 整備工場・業者（BtoB） | 自動車整備士、中古車販売店 | 集客、パーツ在庫の販路確保 | ショップ掲載、パーツ出品、問い合わせ管理 |

## 収益モデル

### 1. フリーミアム（月額サブスクリプション）

個人ユーザー向けの段階的プラン。無料ユーザーがアプリの価値を体験し、有料転換を促す。

#### プラン比較表

| 機能 | Free | Pro（個人） | Business（法人） |
|------|:----:|:---------:|:--------------:|
| 車両登録台数 | 1台 | 5台 | 無制限 |
| 整備記録件数 | 50件/車 | 無制限 | 無制限 |
| OCR（車検証/請求書） | 月3回 | 無制限 | 無制限 |
| PDF出力 | - | 対応 | 対応 |
| AIパーツ提案 | 基本のみ | 詳細+理由付き | 詳細+理由付き |
| SNS投稿 | 対応 | 対応 | 対応 |
| 広告表示 | あり | なし | なし |
| データエクスポート（CSV） | - | 対応 | 対応 |
| 整備ショップ優先問い合わせ | - | - | 対応 |
| 複数スタッフアカウント | - | - | 対応（5名まで） |
| 専用サポート | - | - | 対応 |

#### 想定価格

| プラン | 月額 | 年額（2ヶ月分無料） |
|--------|-----:|------------------:|
| Free | 0円 | 0円 |
| Pro | 580円 | 5,800円 |
| Business | 3,800円 | 38,000円 |

**根拠**:
- Proは競合のみんカラPRO（月額330円）より高いが、整備記録管理+OCR+AIの付加価値で差別化
- Businessはフリーランス整備士や小規模工場が主ターゲット。Googleビジネスプロフィール（月額0円）との差別化は問い合わせ一元管理とAI連携

### 2. マーケットプレイス手数料

ユーザー間のパーツ・車両売買に対する取引手数料。

#### 手数料体系

| 出品カテゴリ | 販売手数料 | 最低手数料 | 上限手数料 |
|------------|--------:|--------:|--------:|
| 車用パーツ・用品 | 8% | 100円 | 5,000円 |
| 中古車両出品（個人） | 3% | 3,000円 | 50,000円 |
| 中古車両出品（業者） | 1.5% + 掲載料 | 5,000円 | - |

**参考**: カーセンサーの中古車掲載料は1台あたり5,000〜30,000円。メルカリの手数料は10%。
本プラットフォームはその中間で、車両特化の付加価値（整備記録連携、互換確認AI）を提供することで正当化する。

#### 決済フロー

```
出品者 → 商品登録 → 購入者がアプリ内決済（Stripe）
→ プラットフォームが一時預かり（エスクロー）
→ 取引完了確認後、手数料を差し引いて出品者に送金（3〜5営業日）
```

#### BtoBパーツ販売（業者向け）

整備工場・パーツショップが在庫をアプリに掲載し、ユーザーに直販できる機能。
月額掲載プラン（後述）と合算課金。

### 3. 整備ショップ掲載費（BtoBサブスクリプション）

整備工場・中古車販売店・カスタムショップ向けのプレミアム掲載プラン。

#### プラン比較表

| 機能 | 無料掲載 | スタンダード | プレミアム |
|------|:-------:|:---------:|:--------:|
| 基本情報掲載 | 対応 | 対応 | 対応 |
| 写真掲載枚数 | 3枚 | 20枚 | 無制限 |
| サービスメニュー掲載 | 3件 | 無制限 | 無制限 |
| 問い合わせ受信 | 月5件 | 無制限 | 無制限 |
| 検索順位ブースト | - | 対応 | 対応（最上位） |
| 「verified」バッジ | - | 対応 | 対応 |
| パーツ出品枠 | - | 50件 | 無制限 |
| 顧客管理（CRM）機能 | - | - | 対応 |
| 月次レポート | - | - | 対応 |

#### 想定価格

| プラン | 月額 | 年額 |
|--------|-----:|----:|
| 無料掲載 | 0円 | 0円 |
| スタンダード | 9,800円 | 98,000円 |
| プレミアム | 29,800円 | 298,000円 |

**根拠**:
- ホットペッパービューティーの美容室掲載料（月額3〜30万円）を参考に、整備工場規模で設定
- 初年度は無料掲載からスタンダードへの転換率20%を目標とする

### 4. パーツ・用品アフィリエイト

AIパーツ提案画面やSNS投稿内のパーツリンクから、外部ECサイトへ誘導し成果報酬を得る。

#### 提携先と報酬率

| 提携先 | カテゴリ | 報酬率 | 備考 |
|--------|---------|------:|------|
| Amazonアソシエイト | 全般 | 1.5〜8% | 即時対応可、商品数が豊富 |
| 楽天アフィリエイト | 全般 | 1〜3% | ポイント還元でユーザー満足度高 |
| カーポイント / オートバックスオンライン | 専門パーツ | 3〜5% | 交渉次第で個別レート |
| モノタロウ | 業務用パーツ | 2〜4% | BtoBユーザー向け |

**重要方針**: AIパーツ提案は「理由付きで複数候補を並べる」設計思想を維持。
アフィリエイトリンクには「PR」表記を明示し、ユーザーの信頼を損なわない。

#### 収益最大化ポイント

- パーツ提案時に「Amazonで見る」「楽天で見る」のボタンを並列表示
- 整備記録に記録されたパーツ名から次回交換提案を自動生成 → 自然なリンク誘導
- 車種×パーツカテゴリの組み合わせでクリック率の高いリンクを最適化

### 5. データ活用（将来・Phase 9以降）

匿名化・集計済みデータを法人向けに提供するBtoBデータビジネス。

**前提**: 個人情報は一切含まない。ユーザーへの事前説明とオプトアウト機能を必ず実装する。

| データ種類 | 提供先 | 収益化モデル | 価値 |
|-----------|--------|------------|------|
| 車種別・地域別の整備頻度・費用統計 | 損害保険会社 | 年間ライセンス料 | 保険料率の精緻化 |
| 消耗品交換タイミングの実態データ | 自動車メーカー・部品メーカー | データ販売 | リコール予防、製品改良 |
| 中古車の整備履歴（匿名・集計） | 中古車査定サービス | API連携料 | 査定精度向上 |
| ユーザーの整備需要予測 | 整備工場チェーン | 見込み客リード | 繁忙期予測、スタッフ計画 |

**推定単価**: 損保会社向けデータライセンス 年間500〜2,000万円（国内主要損保3社と契約が目標）

## 競合との差別化

| 比較軸 | みんカラ | カーセンサー | ガリバー | 楽天Carアプリ | Trust Car Platform |
|--------|:-------:|:-----------:|:-------:|:-----------:|:-----------------:|
| 車両管理・整備記録 | 部分的 | - | - | 対応 | 対応（OCR+AI） |
| SNSコミュニティ | 対応 | - | - | - | 対応 |
| パーツ売買 | 対応 | 一部 | - | - | 対応（予定） |
| AIパーツ提案 | - | - | - | - | 対応（予定） |
| 整備ショップ連携 | - | - | 対応 | 対応 | 対応（予定） |
| 車検アラート | - | - | - | 対応 | 対応 |
| 設計思想 | 趣味SNS特化 | 中古車EC | 中古車買取 | 管理特化 | 全機能統合 |
| 広告モデル | バナー広告主体 | 掲載課金主体 | 買取手数料 | サブスク | フリーミアム+BtoB |

### Trust Car Platformの3つの競争優位

1. 車両データ × SNS × マーケットプレイスの統合
   - 「自分の車種に合うパーツ」を整備記録から自動提案できるのは唯一
   - 競合は縦割り（管理だけ/SNSだけ/売買だけ）

2. 「売り込まない」設計思想による信頼
   - AIは断言しない、事業者はユーザー主導でのみ接触可能
   - 過剰な広告・プッシュ通知で離脱するユーザーを防ぐ

3. 整備記録の継続入力によるデータ資産
   - 使えば使うほど「自分の車のこと」がわかるアプリ
   - スイッチングコストが高く、長期継続につながる

## リスクと対応策

| リスク | 発生確率 | インパクト | 対応策 |
|--------|:-------:|:--------:|--------|
| ユーザー獲得が想定より遅い | 中 | 高 | 整備工場パートナーシップ経由のBtoB2BtoCモデルを強化 |
| Proへの有料転換率が低い | 中 | 高 | フリープランの制限（OCR月3回）を体験後に訴求、トライアル施策 |
| 競合（みんカラ等）の機能追従 | 低 | 中 | AIパーツ提案と整備データ連携の深化で差別化維持 |
| 決済システムの不正利用 | 低 | 高 | Stripeの不正検知 + エスクロー方式でリスク軽減 |
| BtoBショップの掲載離脱 | 中 | 中 | 問い合わせ数・成約数の可視化で成果を示す |
